## 1. 本章在讲什么

这一章的主线很清楚：

1. 从**随机过程**出发；
2. 引出**马尔可夫性质**；
3. 得到**马尔可夫过程 MP**；
4. 加上奖励得到**马尔可夫奖励过程 MRP**；
5. 再加上动作得到**马尔可夫决策过程 MDP**；
6. 在 MDP 中定义**策略、状态价值函数、动作价值函数**；
7. 用**贝尔曼方程**和**蒙特卡洛方法**求价值；
8. 最后引出**占用度量**和**最优策略**。

如果只用一句话概括：

**强化学习研究的核心，就是智能体如何在 MDP 中找到能让长期回报最大的策略。**

---

## 2. 和 MAB 的区别

如果按 MAB 的学习思路来记，这一章最重要的是先搞清楚：**MDP 比 MAB 多了“状态转移”**。

### 2.1 MAB

- 没有状态转移；
- 每次只是在多个臂中选一个；
- 奖励只和当前选择有关；
- 当前动作不会改变未来环境结构。

### 2.2 MDP

- 有状态 $s$；
- 有动作 $a$；
- 选了动作后会转移到下一个状态 $s$；
- 当前动作会影响未来能到达哪些状态，也会影响未来总奖励。

### 2.3 一句话对比

**MAB 是“单步选择问题”，MDP 是“序列决策问题”。**

---

## 3. 随机过程与马尔可夫性质

### 3.1 随机过程

随机过程描述的是一个系统如何随时间变化。  
如果用 $S_t$ 表示时刻 $t$ 的状态，那么整个系统就是一串状态序列：

```math
S_0, S_1, S_2, ...
```

### 3.2 马尔可夫性质

这是整章最基础的定义：

```math
P(S_{t+1}=s' \mid S_t=s, S_{t-1}, \dots, S_0)
=
P(S_{t+1}=s' \mid S_t=s)
```

### 3.3 这个公式怎么理解

它的意思不是“历史不重要”，而是：

**一旦当前状态 $S_t$ 已知，预测未来时就不需要额外再看更早的历史。**

也就是说，当前状态已经把历史里对未来有用的信息“压缩”进来了。

### 3.4 直觉理解

比如下棋时，如果“当前棋盘局面”已经完整包含了所有规则相关信息，那么你下一步怎么走，只需要看当前局面，不需要回忆这盘棋最开始是怎么下过来的。

---

## 4. 马尔可夫过程 MP

马尔可夫过程也叫马尔可夫链，通常写成：

```math
\mathcal{M} = (\mathcal{S}, P)
```

其中：

- $\mathcal{S}$：状态集合
- $P$：状态转移概率

状态转移概率写作：

```math
P(s' \mid s)
```

表示从状态 $s$ 转移到 $s'$ 的概率。

### 4.1 重要性质

对任意状态 `s`，都有：

```math
\sum_{s' \in \mathcal{S}} P(s' \mid s) = 1
```

### 4.2 公式解释

因为从某个状态出发，下一步一定会去某个状态，所以所有可能下一状态的概率加起来必须等于 1。

---

## 5. 马尔可夫奖励过程 MRP

在 MP 基础上加上奖励和折扣因子，就得到 MRP：

```math
(\mathcal{S}, P, R, \gamma)
```

其中：

- $R(s)$：处于状态 $s$ 时的期望奖励
- $\gamma \in [0,1)$：折扣因子

### 5.1 折扣因子 $\gamma$ 的含义

- $\gamma$ 越接近 0：越看重短期奖励
- $\gamma$ 越接近 1：越看重长期奖励

直觉上，未来越远越不确定，所以通常要打折。

---

## 6. 回报 Return

从时刻 $t$ 开始，未来所有奖励的折扣和叫做回报：

```math
G_t = R_{t+1} + \gamma R_{t+2} + \gamma^2 R_{t+3} + \cdots
```

也可以写成：

```math
G_t = \sum_{k=0}^{\infty} \gamma^k R_{t+k+1}
```

### 6.1 这个公式为什么重要

强化学习里真正关心的不是“当前这一步奖励”，而是：

**从现在开始，未来总共能拿到多少奖励。**

这就是 $G_t$。

### 6.2 递推形式

这个公式还有一个极其重要的递推写法：

```math
G_t = R_{t+1} + \gamma G_{t+1}
```

### 6.3 递推形式解释

它表示：

**当前回报 = 当前一步奖励 + 折扣后的下一时刻回报**

这个递推关系是后面所有贝尔曼方程的根。

---

## 7. 价值函数 Value Function

在 MRP 中，状态价值函数定义为：

```math
V(s) = \mathbb{E}[G_t \mid S_t = s]
```

意思是：

**从状态 $s$ 出发时，未来累计回报的期望。**

### 7.1 为什么是“期望”

因为状态转移一般是随机的。  
即使都从同一个状态 `s` 出发，未来走到哪里、拿到多少奖励也可能不同，所以我们看的是平均意义下的长期收益。

---

## 8. 贝尔曼方程

MRP 中最核心的公式之一：

```math
V(s) = R(s) + \gamma \sum_{s' \in \mathcal{S}} P(s' \mid s)V(s')
```

### 8.1 这个公式怎么记

直接背这句话：

**一个状态的价值 = 当前奖励 + 折扣后的下一状态价值的期望。**

### 8.2 为什么成立

由回报递推式

```math
G_t = R_{t+1} + \gamma G_{t+1}
```

两边取条件期望就得到。

### 8.3 公式拆解

- $R(s)$：站在当前状态立刻能拿到的奖励
- $P(s'|s)$：下一步去往不同状态的概率
- $V(s')$：去了下一个状态之后的长期价值
- $\sum P(s'|s)V(s')$：对未来价值求期望
- 再乘上 $\gamma$：表示未来价值要折扣

### 8.4 矩阵形式

把所有状态的价值写成列向量，可以得到：

```math
V = R + \gamma P V
```

移项得：

```math
(I - \gamma P)V = R
```

因此解析解为：

```math
V = (I - \gamma P)^{-1}R
```

### 8.5 这个解析解怎么理解

它说明：

**如果状态数不大，并且转移矩阵 `P` 已知，那么价值函数可以直接通过线性代数一次性求出来。**

### 8.6 局限

- 只能用于状态不大的情况
- 必须知道完整环境模型
- 大规模问题求逆代价高

所以后面才需要动态规划、蒙特卡洛、时序差分。

---

## 9. 马尔可夫决策过程 MDP

MRP 里环境自己演化；MDP 里多了智能体动作。

MDP 定义为：

```math
(\mathcal{S}, \mathcal{A}, P, R, \gamma)
```

其中：

- $\mathcal{S}$：状态集合
- $\mathcal{A}$：动作集合
- $P(s' \mid s,a)$：在状态 $s$ 采取动作 $a$ 后转移到 $s'$ 的概率
- $R(s,a)$：在状态 $s$ 采取动作 $a$ 的期望奖励
- $\gamma$：折扣因子

### 9.1 MDP 的本质

在 MDP 中，未来不只取决于“当前在哪个状态”，还取决于“你现在做什么动作”。

所以强化学习的关键就从“分析过程”变成了“设计策略”。

---

## 10. 策略 Policy

策略记作 $\pi$，表示在某个状态下选动作的规则。

随机策略写作：

```math
\pi(a \mid s)
```

表示在状态 $s$ 下采取动作 $a$ 的概率。

### 10.1 两种策略

#### 确定性策略

每个状态只选一个动作，概率为 1。

#### 随机策略

每个状态输出一个动作概率分布，再按概率采样动作。

### 10.2 为什么 MDP 里策略只依赖当前状态

因为环境满足马尔可夫性质。  
当前状态已经足够描述未来决策所需的信息。

---

## 11. MDP 中的状态价值函数

给定策略 $\pi$ 后，状态价值函数定义为：

```math
V^\pi(s) = \mathbb{E}_\pi [G_t \mid S_t=s]
```

### 11.1 含义

**从状态 $s$ 出发，后续一直按照策略 $\pi$ 行动时，能获得的期望累计回报。**

注意这里的价值已经依赖于策略了，所以写成 $V^\pi(s)$。

---

## 12. 动作价值函数

给定策略 $\pi$，动作价值函数定义为：

```math
Q^\pi(s,a) = \mathbb{E}_\pi [G_t \mid S_t=s, A_t=a]
```

### 12.1 含义

**在状态 $s$ 下先采取动作 $a$，之后再一直遵循策略 $\pi$，所能获得的期望累计回报。**

### 12.2 和状态价值函数的区别

- $V^\pi(s)$：只问“这个状态总体有多好”
- $Q^\pi(s,a)$：进一步问“在这个状态下先做这个动作有多好”

---

## 13. $V^\pi$ 和 $Q^\pi$ 的关系

### 13.1 由 $Q^\pi$ 表示 $V^\pi$

```math
V^\pi(s) = \sum_{a \in \mathcal{A}} \pi(a \mid s) Q^\pi(s,a)
```

### 13.2 解释

状态价值等于该状态下所有动作价值的加权平均，权重就是策略选择各动作的概率。

可以直接理解成：

**状态值 = 按策略选动作后，动作值的期望。**

### 13.3 由 $V^\pi$ 表示 $Q^\pi$

```math
Q^\pi(s,a) = R(s,a) + \gamma \sum_{s'} P(s' \mid s,a)V^\pi(s')
```

### 13.4 解释

动作价值 = 当前动作的即时奖励 + 执行动作后到达各下一状态的未来价值期望。

---

## 14. 贝尔曼期望方程

这是 MDP 里最核心的一组公式。

### 14.1 状态价值形式

```math
V^\pi(s)
=
\sum_a \pi(a \mid s)
\left[
R(s,a) + \gamma \sum_{s'} P(s' \mid s,a)V^\pi(s')
\right]
```

### 14.2 动作价值形式

```math
Q^\pi(s,a)
=
R(s,a) + \gamma \sum_{s'} P(s' \mid s,a)\sum_{a'}\pi(a' \mid s')Q^\pi(s',a')
```

### 14.3 这两条公式在说什么

#### 第一条

在状态 $s$ 下：

1. 先按策略 $\pi(a|s)$ 选动作；
2. 得到即时奖励 $R(s,a)$；
3. 再转移到 $s'$；
4. 然后继续按策略行动。

所以 $V^\pi(s)$ 是“动作概率”和“后续价值”的联合期望。

#### 第二条

如果第一个动作已经固定为 $a$，那后面只需要对：

- 下一状态 $s'$
- 下一步动作 $a'$

继续求期望即可。

### 14.4 一句话记忆

**贝尔曼期望方程 = 把“长期回报”拆成“眼前一步 + 未来期望”。**

---

## 15. MDP 到 MRP 的转化

这一章很重要的一个思想是：

**给定一个策略 $\pi$ 后，MDP 可以转化成 MRP。**

因为动作已经由策略固定成概率分布，所以可以把动作“边缘化”掉。

### 15.1 转化后的奖励函数

```math
R^\pi(s) = \sum_a \pi(a \mid s) R(s,a)
```

### 15.2 转化后的状态转移概率

```math
P^\pi(s' \mid s) = \sum_a \pi(a \mid s) P(s' \mid s,a)
```

### 15.3 为什么这很重要

因为这样我们就可以把“策略固定后的 MDP”当成一个普通的 MRP，然后继续用：

```math
V = (I-\gamma P)^{-1}R
```

来求价值。

### 15.4 直觉

策略 $\pi$ 决定了“在状态 $s$ 下你通常会怎么行动”，于是原来“状态 + 动作”的系统，被压缩成了一个只在状态之间转移的过程。

---

## 16. 蒙特卡洛方法 MC

蒙特卡洛方法的核心思想是：

**不会直接解方程，而是靠大量采样估计期望。**

### 16.1 在 MDP 中怎么用

如果状态价值定义为：

```math
V^\pi(s) = \mathbb{E}[G_t \mid S_t=s]
```

那就可以：

1. 用策略 $\pi$ 采样很多条轨迹；
2. 每次状态 $s$ 出现时，计算从那里开始的回报 $G_t$；
3. 对这些回报求平均。

### 16.2 估计公式

```math
V^\pi(s) \approx \frac{1}{N(s)}\sum_{i=1}^{N(s)} G_i(s)
```

其中：

- $N(s)$：状态 $s$ 出现的次数
- $G_i(s)$：第 $i$ 次出现 $s$ 时对应的回报

### 16.3 增量更新公式

这一条特别常用：

```math
V(s) \leftarrow V(s) + \frac{1}{N(s)}\bigl(G - V(s)\bigr)
```

### 16.4 为什么增量公式成立

设前 $N-1$ 次样本均值为 $V_{N-1}$，第 $N$ 次样本为 $G_N$，则新均值：

```math
V_N
=
\frac{(N-1)V_{N-1}+G_N}{N}
=
V_{N-1} + \frac{1}{N}(G_N - V_{N-1})
```

所以不需要保存所有历史样本，只维护当前均值和计数就够了。

### 16.5 优缺点

优点：

- 不需要知道状态转移概率 $P$
- 不需要知道奖励函数 $R$
- 只靠采样即可

缺点：

- 方差通常较大
- 采样少时误差明显
- 必须等到一段轨迹结束后才能完整算回报

## 17. 占用度量 Occupancy Measure

这一节容易抽象，但其实思想不难：

**不同策略不仅价值不同，它们“会去哪些状态、会做哪些动作”的频率也不同。**

### 17.1 状态访问分布

记 $p_t^\pi(s)$ 为在策略 $\pi$ 下，时刻 $t$ 访问状态 $s$ 的概率。  
状态访问分布本质上描述的是：长期来看，策略会更常去哪些状态。

### 17.2 占用度量定义

占用度量描述状态动作对 $(s,a)$ 被访问的频率，常写作：

```math
\rho^\pi(s,a)
=
(1-\gamma)\sum_{t=0}^{\infty}\gamma^t P(S_t=s, A_t=a \mid \pi)
```

### 17.3 这个公式怎么理解

公式拆开看：

- $P(S_t=s, A_t=a \mid \pi)$：第 $t$ 步恰好访问到状态动作对 $(s,a)$ 的概率
- $\gamma^t$：越后面的访问权重越小
- $(1-\gamma)$：归一化因子

所以占用度量本质上是在描述：

**在折扣意义下，一个策略有多常访问某个状态动作对。**

### 17.4 与状态访问分布的关系

```math
\rho^\pi(s,a) = \rho^\pi(s)\pi(a \mid s)
```

意思是：

状态动作对被访问的频率 = 状态被访问的频率 × 在该状态选择该动作的概率。

### 17.5 为什么这一节重要

因为很多强化学习和模仿学习方法，本质上都在比较两个策略的占用度量是否接近。

你可以把它理解成：

**比较两个策略，不只是看最终奖励，还可以看它们“行为轨迹分布”是否相似。**

---

## 18. 最优策略

强化学习最终目标是找到一个最优策略 $\pi^*$，使得任意状态下的价值都尽可能大。

### 18.1 最优状态价值函数

```math
V^*(s) = \max_\pi V^\pi(s)
```

表示在状态 `s` 下，所有策略里能达到的最大价值。

### 18.2 最优动作价值函数

```math
Q^*(s,a) = \max_\pi Q^\pi(s,a)
```

表示在状态 `s` 下先做动作 `a`，之后再采取最优策略时，所能达到的最大价值。

### 18.3 二者关系

```math
V^*(s) = \max_a Q^*(s,a)
```

### 18.4 解释

如果你已经知道每个动作有多好，那状态的最优价值当然就是选其中最好的那个动作。

---

## 19. 贝尔曼最优方程

这是整章最重要的公式之一，一定要会。

### 19.1 状态价值形式

```math
V^*(s) = \max_a \left[ R(s,a) + \gamma \sum_{s'} P(s' \mid s,a)V^*(s') \right]
```

### 19.2 动作价值形式

```math
Q^*(s,a)
=
R(s,a) + \gamma \sum_{s'} P(s' \mid s,a)\max_{a'}Q^*(s',a')
```

### 19.3 和贝尔曼期望方程的区别

贝尔曼期望方程里是：

```math
\sum_a \pi(a \mid s)
```

也就是“按策略求平均”。

贝尔曼最优方程里是：

```math
\max_a
```

也就是“不再求平均，而是直接挑最优动作”。

### 19.4 一句话理解

**最优价值 = 当前能选的动作里，那个“即时奖励 + 未来最优价值”最大的动作对应的结果。**

### 19.5 为什么这是强化学习的核心

因为它把“找最优策略”变成了一个递推问题。  
后面的动态规划、Q-learning、DQN，本质上都在围绕这个思想做逼近。

```math
G_t = \sum_{k=0}^{\infty}\gamma^k R_{t+k+1}
```

```math
G_t = R_{t+1} + \gamma G_{t+1}
```


```math
V(s) = R(s) + \gamma \sum_{s'}P(s' \mid s)V(s')
```

```math
V = (I-\gamma P)^{-1}R
```


```math
Q^\pi(s,a) = \mathbb{E}_\pi [G_t \mid S_t=s,A_t=a]
```

```math
V^\pi(s) = \sum_a \pi(a \mid s)Q^\pi(s,a)

```

```math
V^\pi(s)
=
\sum_a \pi(a \mid s)\left[R(s,a)+\gamma\sum_{s'}P(s' \mid s,a)V^\pi(s')\right]
```

```math
Q^\pi(s,a)
=
R(s,a)+\gamma\sum_{s'}P(s' \mid s,a)\sum_{a'}\pi(a' \mid s')Q^\pi(s',a')
```

### 20.11 MDP 转 MRP

```math
R^\pi(s)=\sum_a \pi(a \mid s)R(s,a)
```

```math
P^\pi(s' \mid s)=\sum_a \pi(a \mid s)P(s' \mid s,a)
```

### 20.12 蒙特卡洛增量更新

```math
V(s) \leftarrow V(s) + \frac{1}{N(s)}(G - V(s))
```

### 20.13 占用度量

```math
\rho^\pi(s,a)
=
(1-\gamma)\sum_{t=0}^{\infty}\gamma^t P(S_t=s,A_t=a \mid \pi)
```

### 20.14 最优状态价值函数

```math
V^*(s)=\max_\pi V^\pi(s)
```

### 20.15 最优动作价值函数

```math
Q^*(s,a)=\max_\pi Q^\pi(s,a)
```

### 20.16 贝尔曼最优方程

```math
V^*(s)=\max_a \left[R(s,a)+\gamma\sum_{s'}P(s' \mid s,a)V^*(s')\right]
```

```math
Q^*(s,a)
=
R(s,a)+\gamma\sum_{s'}P(s' \mid s,a)\max_{a'}Q^*(s',a')
```

## 21. 这章最容易混淆的点

### 21.1 $V(s)$ 和 $V^\pi(s)$ 的区别

- $V(s)$：通常用于 MRP
- $V^\pi(s)$：用于 MDP，强调价值依赖策略

### 21.2 $V^\pi(s)$ 和 $Q^\pi(s,a)$ 的区别

- $V^\pi(s)$ 是状态整体的好坏
- $Q^\pi(s,a)$ 是“先做动作 $a$”后的好坏

### 21.3 贝尔曼期望方程和贝尔曼最优方程的区别

- 期望方程：对策略做加权平均
- 最优方程：直接取最大值

### 21.4 MRP 和 MDP 的区别

- MRP 没有动作
- MDP 有动作和策略

### 21.5 MAB 和 MDP 的区别

- MAB 只考虑“当前选哪个臂”
- MDP 还要考虑“这一步会怎样影响未来状态”

## 22. 考试 / 面试式总结

如果要你用几句话概括这一章，可以这样说：

1. 马尔可夫性质说明未来只依赖当前状态，不依赖完整历史。
2. 马尔可夫过程在此基础上描述状态转移。
3. 加奖励得到 MRP，可以定义回报和状态价值函数。
4. MRP 的价值满足贝尔曼方程，矩阵形式可写成 $V = (I-\gamma P)^{-1}R$。
5. 加动作得到 MDP，策略 $\pi(a|s)$ 决定在每个状态选什么动作。
6. MDP 中有状态价值函数 $V^\pi(s)$ 和动作价值函数 $Q^\pi(s,a)$。
7. 它们满足贝尔曼期望方程。
8. 给定策略后，MDP 可以边缘化为 MRP。
9. 价值也可以通过蒙特卡洛采样估计。
10. 强化学习的最终目标是找到满足贝尔曼最优方程的最优策略。
