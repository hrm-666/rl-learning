# 强化学习：策略迭代与价值迭代学习笔记

---

# 一、强化学习中的核心目标

强化学习（Reinforcement Learning, RL）的核心目标：

> 让智能体（Agent）通过与环境交互，学习一个长期收益最大的策略。

强化学习并不只是“当前奖励最大”。

真正优化的是：

# 长期累计回报（Return）

即：

G_t=r_t+\gamma r_{t+1}+\gamma^2 r_{t+2}+\cdots

其中：

* (r_t)：当前奖励
* (\gamma)：折扣因子

---

# 二、状态价值函数与动作价值函数

---

# 1. 状态价值函数

状态价值函数：

V^{\pi}(s)

表示：

> 在状态 (s) 下，
> 按照策略 (\pi) 持续行动，
> 最终能获得的长期累计奖励期望。

---

## 核心理解

状态价值函数本质：

# “未来收益预测器”

价值越高：

说明这个状态越值得到达。

---

# 2. 动作价值函数

动作价值函数：

Q^{\pi}(s,a)

表示：

> 在状态 (s) 下，
> 先执行动作 (a)，
> 后续继续按照策略 (\pi) 行动，
> 最终获得的长期收益。

---

# 三、Bellman 方程（核心）

Bellman 方程是强化学习最核心的思想。

---

# 1. Bellman 期望方程

固定策略下：

V^{\pi}(s)=\sum_a\pi(a|s)Q^{\pi}(s,a)

即：

# 状态价值 = 所有动作价值的加权平均

其中：

[
\pi(a|s)
]

表示：

> 在状态 (s) 下采取动作 (a) 的概率。

---

进一步展开：

V^{\pi}(s)=\sum_a\pi(a|s)\left[r+\gamma\sum_{s'}P(s'|s,a)V^{\pi}(s')\right]

---

# 核心直觉

Bellman 方程本质：

# 当前价值

=

# 当前奖励

*

# 未来价值

---

即：

未来收益会不断向前传播。

因此：

* 离终点越近
* 价值越高

即使当前状态本身没有奖励。

---

# 四、Bellman 最优方程

最优状态价值函数：

V^*(s)=\max_a Q^*(s,a)

其含义：

> 在所有动作中，
> 选择未来收益最大的动作。

---

这里与 Bellman 期望方程最大的区别：

---

## Bellman 期望方程

固定策略：

```text
按照当前策略求期望
```

---

## Bellman 最优方程

最优策略：

```text
直接选择最优动作
```

---

# 五、策略迭代（Policy Iteration）

策略迭代包括两部分：

```text
策略评估
→
策略提升
```

不断循环直到收敛。

---

# 1. 策略评估（Policy Evaluation）

目标：

> 已知当前策略 (\pi)，
> 求该策略对应的价值函数 (V^\pi(s))。

---

## 迭代公式

V^{k+1}(s)=\sum_a\pi(a|s)\left[r+\gamma\sum_{s'}P(s'|s,a)V^k(s')\right]

---

## 为什么需要反复迭代？

因为：

Bellman 方程是：

# 递归不动点方程

真实价值函数：

[
V^\pi
]

满足：

V^{\pi}=T^{\pi}V^{\pi}

因此：

策略评估实际上是在：

# 不断逼近 Bellman 不动点。

---

# 2. 策略提升（Policy Improvement）

策略提升目标：

> 根据当前价值函数，
> 重新选择更优动作。

---

## 更新公式

\pi_{new}(s)=\arg\max_a Q^{\pi}(s,a)

---

## 核心思想

如果当前价值函数已经告诉我们：

```text
某个动作未来收益更高
```

那么：

以后就更应该选择它。

---

# 3. 为什么策略提升不会变差？

因为：

新策略始终选择：

# 旧价值函数下更优动作。

因此：

新策略至少不会比旧策略更差。

这就是：

# Policy Improvement Theorem

---

# 4. 策略迭代完整流程

```text
初始化策略 π

while True:

    1. 固定 π
       进行策略评估
       求 Vπ

    2. 根据 Vπ
       贪心更新策略

    3. 若策略不再变化
       则停止
```

---

# 六、价值迭代（Value Iteration）

---

# 1. 核心思想

价值迭代：

# 不进行完整策略评估。

而是：

# 一边更新价值

# 一边直接做贪心

---

# 更新公式

V_{k+1}(s)=\max_a\left[r+\gamma\sum_{s'}P(s'|s,a)V_k(s')\right]

---

# 2. 与策略迭代的区别

---

## 策略迭代

```text
完整评估
→
统一提升
```

优点：

* 收敛稳定

缺点：

* 每次评估计算量大

---

## 价值迭代

```text
边更新
边贪心
```

优点：

* 更快

缺点：

* 每次更新更“激进”

---

# 七、Cliff Walking 悬崖漫步环境

---

# 1. 环境结构

```text
S C C C C C C C C C C G
```

其中：

* S：起点
* C：悬崖
* G：终点

---

# 2. 奖励设计

---

## 普通移动

```text
reward = -1
```

作用：

# 惩罚拖延

迫使 Agent 找最短路径。

---

## 掉入悬崖

```text
reward = -100
```

作用：

# 强烈惩罚危险行为。

---

# 3. 为什么价值全是负数？

因为：

每走一步：

```text
-1
```

因此：

Agent 本质是在：

# “尽量少亏”

---

例如：

```text
Q右 = -3
Q上 = -5
```

虽然都为负数：

但：

[
-3 > -5
]

因此：

向右更优。

---

# 八、折扣因子 gamma 的作用

---

# 1. gamma 接近 1

```python
gamma = 0.99
```

表示：

# 更关注长期收益。

---

# 2. gamma = 0

此时：

Q(s,a)=r

Agent：

# 只看当前奖励。

变成：

# 纯贪心策略。

不再考虑未来。

---

# 九、价值传播（最重要的直觉）

强化学习最核心的本质：

# “价值传播”

---

例如：

终点奖励：

```text
+100
```

那么：

* 离终点近的状态价值高
* 远处状态价值低

奖励会：

# 从未来向前传播。

---

因此：

强化学习能够：

* 长期规划
* 延迟奖励
* 学习路径选择

---

# 十、Reward Design 的重要性

强化学习学到的行为：

# 完全由奖励函数决定。

---

例如：

---

## 悬崖奖励 = -100

Agent：

```text
宁可绕远
也不靠近悬崖
```

---

## 悬崖奖励 = -2

Agent：

```text
可能直接冒险走近路
```

因为：

总体长期收益可能更高。

---

因此：

# Reward Design 决定 Agent 行为。

---

# 十一、策略迭代与价值迭代真正学到了什么？

表面上：

* 策略迭代学策略
* 价值迭代学价值

但更深层：

它们真正学习的是：

# “哪些状态更值得到达”

以及：

# “哪些动作会把你带向高价值状态”

---

因此：

强化学习真正学习的是：

# 长期收益如何在状态之间传播。
